# ADR-003 — NumPy para Matemática do Transform

**Status:** Aceito • **Data:** 2026-07-01

---

## Contexto

O `Transform` precisa armazenar e operar sobre vetores de posição, rotação e
escala em 2D e 3D. Python puro (listas, tuplas) é lento para operações
vetorizadas repetidas a cada frame.

## Decisão

Os vetores internos do `Transform` são `np.array(..., dtype=np.float32)`.
Todas as operações aritméticas (translate, rotate, model matrix) usam NumPy.

```python
self._position = np.array([x, y, z], dtype=np.float32)
self._position += np.array([dx, dy, dz], dtype=np.float32)  # vetorizado
```

A matriz modelo (`get_model_matrix`) é uma multiplicação de matrizes 4x4 NumPy,
compatível com ModernGL para o pipeline 3D.

## Consequências

**Positivas:**
- Operações vetorizadas rápidas (C sob o capô)
- Compatibilidade direta com ModernGL (envia `np.ndarray` para shaders)
- Suporte nativo a 2D (z=0) e 3D com o mesmo tipo

**Negativas:**
- NumPy como dependência obrigatória (mas já é usado no projeto inteiro)
- Acessar `t.x` retorna `float(self._position[0])` — conversão pequena por acesso

## Alternativas Consideradas

- PyGLM (GLM para Python): excelente API, mas adiciona dependência extra
- Listas Python puras: simples, mas sem vetorização e incompatível com ModernGL
- `pygame.math.Vector2/3`: limitado a 2D/3D separados, sem matriz modelo
