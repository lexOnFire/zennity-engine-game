# ADR-001 — engine/core/ como Pacote Canônico

**Status:** Aceito • **Data:** 2026-07-01

---

## Contexto

O projeto nasceu com todo o núcleo espalhado em arquivos soltos em `engine/*.py`
(`engine/core.py`, `engine/component.py`, `engine/scene_manager.py`, etc.).
Com o crescimento do projeto, ficou impossível saber onde cada classe deveria
evoluir e manter a retrocompatibilidade de imports existentes.

## Decisão

Todo o núcleo evolui exclusivamente dentro do pacote `engine/core/`.
Os arquivos `engine/*.py` existentes tornam-se shims de compatibilidade que
re-exportam de `engine/core/` e emitem `DeprecationWarning` quando importados
diretamente.

O import canônico para todo código novo é:
```python
from engine.core import Scene, GameObject, Component, Transform
from engine.core import Application, SceneManager, Engine
```

## Consequências

**Positivas:**
- Um único lugar onde o núcleo evolui
- Imports previsíveis e documentados
- Possível remover os shims numa versão futura sem quebrar código novo

**Negativas:**
- Temporariamente dois arquivos para cada classe (canônico + shim)
- Desenvolvedores com código legado verão `DeprecationWarning` até migrarem

## Alternativas Consideradas

- Manter tudo em `engine/*.py` e crescer no mesmo lugar (rejeitado: caos de imports)
- Renomear o pacote para `zennity/` (rejeitado: quebra retrocompat total)
