# ADR-004 — Pygame/SDL2 como Backend de Janela e Input

**Status:** Aceito • **Data:** 2026-07-01

---

## Contexto

A engine precisa de um backend para criar janela, processar eventos de teclado/
mouse/gamepad e renderizar frames. Para o pipeline 2D, também precisa de
operações de blit, superfícies e fontes.

## Decisão

Pygame (binding SDL2) é o backend de janela e input. O pipeline 2D usa
`pygame.Surface` e `pygame.draw`. O pipeline 3D usa ModernGL com contexto
OpenGL criado dentro da janela Pygame.

A `Application` e o `Engine` são agnósticos ao backend — eles delegam a criação
da janela para um objeto `Window` interno que encapsula Pygame.

## Consequências

**Positivas:**
- Pygame é bem conhecido na comunidade Python
- SDL2 é maduro, multiplataforma e mantido ativamente
- Integração com ModernGL é direta via contexto OpenGL na janela Pygame
- Sem dependências pesadas (Qt, wx) para o runtime

**Negativas:**
- Pygame não suporta Vulkan ou Metal diretamente
- Para VR futura, será necessário trocar o backend de janela (ex: GLFW)

## Alternativas Consideradas

- GLFW direto: melhor para OpenGL puro, mas sem pipeline 2D integrado
- PySide6 para o runtime: overkill, Qt é para o editor, não para o jogo
- Arcade (biblioteca baseada em pyglet): boa, mas acopla render ao runtime
